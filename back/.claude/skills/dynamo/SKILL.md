## Single-table design

All entities share one DynamoDB table (`DYNAMO_TABLE`, default `data`), with `pk` (String) and `sk` (String) as keys.

### Key schema by entity type

| Entity              | pk                                | sk                  |
|---------------------|-----------------------------------|---------------------|
| ProductType         | `PT#${producerAccountId}`         | `${productTypeId}`  |
| Organization        | `ORGANIZATION`                    | `${orgId}`          |
| Server              | `SERVER`                          | `${serverId}`       |
| OrganizationRequest | `ORGREQ`                          | `${requestId}`      |
| ProducerAccount     | `PA#${organizationId}`            | `${producerAccountId}` |
| Change              | `CHANGE#${entityType}#${scopeId}` | `${entityId}`       |

GSI `by_cursor`: PK=`change_pk` (`${entityType}#${scopeId}`), SK=`cursor` — sparse, only Change items.
GSIs `by_organization_name`, `by_admin_email`: KEYS_ONLY — sparse, only OrganizationRequest items.

### Adding a new synced entity

New entity prefix convention: pick a short uppercase prefix (e.g., `MY#${scopeId}`).

## DAO pattern — DynamoDbMapper (preferred for typed entities)

Use the AWS DynamoDB Mapper for entities with clean read paths:

```kotlin
@DynamoDbItem
internal data class MyEntityDynamo(
    @DynamoDbPartitionKey val pk: String,
    @DynamoDbSortKey val sk: String,
    val name: String,
    val description: String? = null,
) {
    constructor(model: MyEntity) : this(
        pk = "MY#${model.producerAccountId.id}",
        sk = model.myEntityId.id,
        name = model.name,
        description = model.description,
    )

    fun toAttributeValueMap(): Map<String, AttributeValue> = buildMap {
        put("pk", AttributeValue.S(pk))
        put("sk", AttributeValue.S(sk))
        put("entity_type", AttributeValue.S("MyEntity"))   // include for future filtering
        put("name", AttributeValue.S(name))
        description?.let { put("description", AttributeValue.S(it)) }
    }
}

internal fun MyEntity(dynamo: MyEntityDynamo): MyEntity = MyEntity(
    myEntityId = dynamo.sk.toId(),
    producerAccountId = dynamo.pk.removePrefix("MY#").toId(),
    name = dynamo.name,
    description = dynamo.description,
)
```

Add the mapper table to `DynamoClient`:
```kotlin
val myEntityTable = mapper.getMyEntityDynamoTable(table)  // generated by KSP
```

## DAO pattern — raw client (for entities not using mapper)

```kotlin
@Single(createdAtStart = true, binds = [MyEntityDAO::class])
internal class MyEntityDynamoDAO(private val client: DynamoClient) : MyEntityDAO {

    override suspend fun put(entity: MyEntity, change: Change) {
        client.client.transactWriteItems(
            TransactWriteItemsRequest {
                transactItems = listOf(
                    TransactWriteItem { put = Put {
                        tableName = client.table
                        item = MyEntityDynamo(entity).toAttributeValueMap()
                    }},
                    TransactWriteItem { put = Put {
                        tableName = client.table
                        item = change.toAttributeValueMap()   // defined in ChangeDynamoDAO.kt
                    }},
                )
            }
        )
    }

    override suspend fun delete(id: Id<MyEntity>, scopeId: Id<ProducerAccount>, change: Change) {
        client.client.transactWriteItems(
            TransactWriteItemsRequest {
                transactItems = listOf(
                    TransactWriteItem { delete = Delete {
                        tableName = client.table
                        key = mapOf(
                            "pk" to AttributeValue.S("MY#${scopeId.id}"),
                            "sk" to AttributeValue.S(id.id),
                        )
                    }},
                    TransactWriteItem { put = Put {
                        tableName = client.table
                        item = change.toAttributeValueMap()
                    }},
                )
            }
        )
    }
}
```

**Always use `TransactWriteItemsRequest`** for any write that involves a `Change` record — entity + change must be atomic.

## Change record

`Change.toAttributeValueMap()` is defined in `ChangeDynamoDAO.kt`. Fields written:
- `pk`: `CHANGE#${entityType}#${scopeId}`
- `sk`: `${entityId}`
- `change_pk`: `${entityType}#${scopeId}` (GSI key)
- `cursor`, `entity_type`, `producer_account_id`, `op`, `produced_at`, optional `payload`

Never construct the Change attribute map manually — call `change.toAttributeValueMap()`.

## Troubleshooting — container started from a worktree

`DynamoTestInfra.ensureStarted()` starts the container via `docker compose -f src/test/docker/docker-compose.yml up -d` **from the current working directory at test time**. When tests run from an isolated worktree (`.claude/worktrees/agent-*/`), the DynamoDB volume is bound to the worktree path. Once the worktree is deleted, the container keeps running but its volume path is a ghost — DynamoDB Local loops indefinitely on `unable to open database file`.

Symptom — repeated lines in `docker logs dynamodb-local`:
```
WARNING: [sqlite] SQLiteQueue[shared-local-instance.db]: stopped abnormally, reincarnating in 3000ms
SEVERE: [sqlite] error running job queue — unable to open database file
```

Verify: `docker inspect dynamodb-local --format='{{json .HostConfig.Binds}}'` — if the path contains `.claude/worktrees/`, the container is broken.

Fix: restart from the module directory.
```bash
docker rm -f dynamodb-local
cd back/persistence/dynamo
docker compose -f src/test/docker/docker-compose.yml up -d
```
